# Build Checklist

## Build Preferences

- **Build mode:** Step-by-step
- **Comprehension checks:** Yes
- **Git:** Commit after each item with message: "Complete step N: [title]"
- **Verification:** Yes — run the app and confirm acceptance criteria after each item before moving on
- **Check-in cadence:** Learning-driven — talk through what was built and why at each step

---

  Acceptance: From PRD: "Confirming unlocks ALL blocked apps for 15 minutes (home screen) or the tapped app only (shield). A notification fires when 3 minutes of unlock time remain. The secondary countdown disappears when the unlock window ends."

- [x] **8. DeviceActivityMonitor extension — session enforcement and unlock expiry re-lock**
  Spec ref: `spec.md > Targets > DeviceActivityMonitor Extension`, `spec.md > Architecture Overview > Blocked app tap — full data flow`
  What to build: Implement `MonitorExtension.swift` in the DeviceActivityMonitor target. Handle four event types: (1) session start (scheduled) → read `blockedApps` from SharedStore, activate ManagedSettings blocking for all apps, write `isSessionActive = true` and `sessionEndTime`; (2) session end (scheduled) → call `relockAll()`, write `isSessionActive = false`, clear `sessionEndTime`, write a `SessionLog` entry to SwiftData; (3) per-app unlock expiry → re-block that app, remove its entry from `individualUnlockExpiries`; (4) all-apps unlock expiry → call `relockAll()`, clear `allAppsUnlockExpiry`. Wire up DeviceActivity schedule registration in `BlockingService` or at onboarding Finish — register a recurring `DeviceActivitySchedule` from SwiftData schedule data.
  Acceptance: From PRD: "When the 15-minute unlock expires, all blocked apps re-lock and the user is kicked out of any currently open blocked app. The secondary countdown disappears when the unlock window ends. Schedule changes, app list changes, and friction level changes are non-interactive during an active session."
  Verify: Configure a test schedule starting 2 minutes from now. Wait for the session to start — confirm `isSessionActive` flips to true in SharedStore and HomeView updates (lock closes, countdown starts). Trigger a 15-minute unlock, wait for expiry — confirm apps re-lock and `allAppsUnlockExpiry` is cleared. (Full verification of app ejection requires Family Controls entitlement.)
